네이밍 규칙

네이밍 규칙은 레이아웃, 객체, 이미지, 폴더, 파일의 이름을 작성하는 규칙이다. 이해하기 쉬운 이름으로 작성해야 코드를 쉽게 파악할 수 있습니다. 단, 모바일과 같이 성능, 네트워크 속도를 고려해야 하는 경우 축약된 형태로 작성할 수 있습니다.

HTML 코드 작성 규칙

A. 들여쓰기
HTML 코드를 작성할 때 코드의 가독성을 높이기 위하여 왼쪽 첫 번째 열부터 오른쪽으로 일정한 간격만큼 띄어 쓴다. 들여쓰기를 하면 전 체 HTML 구조를 쉽게 파악할 수 있습니다.
B. 빈 줄
HTML 코드의 빈 줄은 코드 그룹의 영역을 표시하기 위하여 사용합니다.
C. DTD 및 인코딩
DTD(Document Type Definition)는 SGML(Standard Generalized Markup Language) 계열 마크업 언어의 문서 타입을 정의하는 것으로서, 해당 HTML 문서가 어떤 버전의 HTML로 작성되었는지, 어떤 규칙으로 내용을 기술하고 어떤 엘리먼트와 애트리뷰트, 애트리뷰트값을 지 정할 수 있는지 정의합니다. 또한 인코딩을 선언하여 문서에서 사용되는 문자 코드 세트를 지정합니다. DTD와 인코딩 선언은 HTML 문서가 브 라우저에서 바르게 해석될 수 있도록 합니다.
D. 주석
HTML 코드의 주석은 코드 그룹을 구분하거나, 코드의 수정과 삭제를 표시하거나, 개발자가 참고해야 하는 사항을 기술합니다.

HTML 엘리먼트 작성 규칙

HTML 엘리먼트 작성 규칙은 반드시 선언해야 하는 애트리뷰트와 선택적 사용이 가능한 애트리뷰트에 대한 내용을 기술하고 선언 순서를 제시하여 코드 품질을 유지합니다.

CSS 코드 작성 규칙

A. 들여쓰기
CSS 코드는 들여쓰기를 하지 않습는다.
B. 공백
CSS 코드는 공백을 최소화합니다.
C. 빈 줄
CSS 코드의 빈 줄은 코드 그룹의 영역을 표시하기 위하여 사용합니다.
D. 줄 바꿈
CSS 코드의 가독성을 위한 줄 바꿈은 하지 않습는다.
E. 인코딩
CSS의 인코딩은 HTML의 인코딩과 동일하게 선언하여 HTML 문서가 브라우저에서 바르게 해석될 수 있도록 합니다.
F. 선택자
선택자 버그가 발생하지 않도록 사용 규칙을 준수합니다.
G. 속성
속성 선언 순서를 준수하여 개발자가 코드를 쉽게 파악할 수 있도록 하며, CSS 코드 최적화를 위해 속성 값을 축약하여 사용하고 약식 속 성을 허용 범위에 맞게 사용합니다.
H. z-index
z-index 속성 값을 범위에 맞게 사용하여 객체가 브라우저에서 바르게 표현되도록 합니다.
I. 핵
크로스 브라우징을 위해 제시된 핵(hack)에 한해 최소한의 사용을 허용합니다.
J. CSS 선언 타입
상황에 알맞은 CSS 선언 타입을 선택합니다.
K. 주석
CSS 코드의 주석은 코드 그룹을 구분하거나, 코드의 수정과 삭제를 표시하거나, 개발자가 참고해야 하는 사항을 기술합니다.
