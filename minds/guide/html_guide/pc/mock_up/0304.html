파일 및 폴더 네이밍 규칙

A. 이미지 네이밍
  • 산출되는 모든 HTML 파일과 폴더를 분류해야 할 경우, 페이지 마스터 파일에서 정의된 페이지 코드와 폴더 코드를 그대로 사용합니다. 단, '코드_이름' 규칙 중 마크업 산출물의 경우만 코드만으로 파일 명을 정의할 수 있으며 뒤의 '이름(영문만 허용) '은 선택에 따라 정의할수 있습니다.
  • 정의된 코드가 없는 프로젝트의 경우, 표 2-8 파일 및 폴더 네이밍 규칙을 따른다.
  • 모바일의 경우 서비스 및 디바이스 환경에 따라 변경될 수 있으므로 표 2-9 를 참조합니다.

B. 파일 및 폴더 네이밍 조합
  • HTML 파일은 페이지 마스터 파일에서 정의한 페이지 코드를 그대로 사용하거나, '코드_이름(영문만 허용)'으로 조합합니다.
  • 페이지 마스터 파일이 없는 경우 '메뉴이름_의미_상태' 순서로 조합합니다.
  • CSS 파일은 '서비스이름'을 맨 앞으로 하여 조합합니다.
