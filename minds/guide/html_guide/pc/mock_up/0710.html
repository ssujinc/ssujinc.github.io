변수 선언 시 "const", "let" 키워드 사용

"var"는 함수 스코프를 가진다. 코드 실행 전에 해당 변수가 hoist되므로 선언부보다 위에서 변수를 호출하여도 에러가 발생하지 않습는다. 블 록 스코프에서 "var"를 사용해도 블록 스코프 밖의 함수 스코프에 변수가 hoist되어 블록 스코프 밖에서 접근이 가능합니다. 이는 Undefined 된 변수의 프로퍼티에 접근할 때, 에러가 발생하는 잠재적인 원인이 됩니다. 반면에, "const"와 "let"은 개발자가 예상한대로 동작합니다. 이 키 워드를 통해 선언한 변수는 블록 스코프를 가진다. TDZ의 혜택을 받아 같은 스코프 내에서 선언 전에 호출하면 에러가 발생합니다.

표 6-36 조건문 작성 예
잘못된 예 올바른 예
var foo = 'foo';
var bar = 'bar';
...
bar = 'var';
const foo = 'foo';
let bar = 'bar';
...
bar = 'var';

"const"를 사용해 변수를 선언하되, 할당 후 값이 변하면 "let"을 사용합니다.

"require", "module.exports“ 지향

자바스크립트에서의 모듈 패턴은 전역에서 특정 변수 영역을 보호하기 위해 사용됩니다. AMD, CommonJS 등의 모듈러가 보편화됨에 따라, ES6에서는 모듈 패턴을 지원하는 키워드인 import, export를 도입했다. 다만, 이 기능은 지원하는 브라우저의 점유율이 매우 낮기에 ES6 문 법을 사용하기 위해서는 트랜스파일러를 사용해야 합니다. import, export를 사용할 수 있는 환경이라면, require, module.exports 보다는 import, export를 사용하라. 사용할 수 없다면, "require", "module.exports"를 사용하라.

표 6-37 조건문 작성 예
잘못된 예 올바른 예
/* Top of file */
const StyleGuide = require('./StyleGuide');
module.exports = StyleGuide.es6;
/* Top of file */
import StyleGuide from './StyleGuide';
export default StyleGuide.es6;

/* Top of file */
import { es6 } from './StyleGuide';
export default es6;

Import문 선언 위치

import문은 hoist 비용을 고려하여 코드의 맨 위로 작성합니다.

표 6-38 조건문 작성 예
잘못된 예 올바른 예
import foo from 'foo';
foo.init();

import bar from 'bar';
/* Top of file */
import foo from 'foo';
import bar from 'bar';

foo.init();

제너레이터 사용 위치 주의점

제너레이터의 *의 위치가 바르지 않으면 트랜스파일링 되지 않습는다. 따라서 가급적 제너레이터를 사용하지 않아야 합니다. 꼭 사용해야 한다면 *의 위치에 주의합니다. 반드시 function 바로 뒤에 써야 합니다.

표 6-39 조건문 작성 예
잘못된 예 올바른 예
function * foo() {
    ...
}

const bar = function * () {
    ...
};

const baz = function *() {
    ...
};

function *foo() {
    ;...
}

// very bad function
* foo() {
    ...
}

const wat = function
* () {
    ...
};
function* foo() {
    ...
}

const foo = function* () {
    ...
};
