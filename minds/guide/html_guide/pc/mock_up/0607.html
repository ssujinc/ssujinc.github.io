기본 규칙

A. 공통 선택자 사용 지양

공통 선택자 '*'는 웹 페이지의 성능을 떨어뜨리고, Internet Explorer에서는 주석까지 영향을 받을 수 있으므로 사용하지 않습는다.

B. id 선택자 범위

id 선택자는 2.2 id 및 class 네이밍 규칙에서 정의한 레이아웃(wrap, header, container, content, footer)에만 사용합니다.

C. 교차속성 사용

인터넷 익스플로러 6.0 브라우저 환경에서 교차속성 오류가 발생하므로 id와 class, class와 class 간의 교차속성을 사용하지 않습는다. 모바일 에서는 IE6의 제약을 받지 않으므로 사용할 수 있습니다.

표 5-9 교차속성 사용 예
잘못된 예 올바른 예
.class{width:980px}
.class.bg{background:yellow}
.class.bg01{background:green} 		.class{width:980px}
.bg{background:yellow}
.bg01{background:green}
참고
  • 교차속성이란 두 선택자의 조합에 의해 속성이 부여되는 경우를 의미합니다.
  • 아래 예에서 background:yellow 속성은 .class와 .bg가 동시에 선언될 때만 의미가 있으며, background:green 속성은 .class와 .bgv1이 동 시에 선언될 때만 의미가 있습니다.
.class{width:980px}
.class.bg{background:yellow}
.class.bg01{background:green}
  • 인터넷 익스플로러 6.0에서는 id와 class 간 교차속성을 사용할 경우 두 번째 선언된 교차속성은 무시하는 버그가 있으며, class와 class 간의 교차속성을 잘못 해석하는 버그가 있습니다. 이와 같은 버그를 방지하기 위해 교차속성을 사용하지 않도록 합니다.
  • 교차속성을 사용하지 않으면 코드양을 줄이고 각 class 간 독립성을 유지할 수 있습니다.
