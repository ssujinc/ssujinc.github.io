  • 전역 구조화 엘리먼트

참고사항

  • 이 문서는 마크업 개발자가 소스 코드 작성 시에 따라야 할 규칙을 기술합니다.
  • 이 문서는 마크업 언어를 다루는 개발자를 대상으로 합니다.

기본 규칙

A. <html>

class 애트리뷰트는 선언하지 않습는다.

<html lang="ko">

XHTML DTD 선언 시에는 다음과 같이 lang 애트리뷰트를 선언합니다.

<html xmlns="http://www.w3.org/1999/xhtml" xml:lang="ko" lang="ko">

lang 애트리뷰트는 User Agent가 언어를 올바로 해석할 수 있게 도와주며, 검색과 음성 장치(speech synthesizers)에 활용됩니다. 언어 코드 는 모든 엘리먼트에 사용할 수 있지만 HTML 엘리먼트에 해당 문서의 주 언어 코드만 선언합니다.
HTML 4.01 Specification, XHTML 1.0 Specification에서 자연어는 RFC 1766에 명시된 언어 코드를 사용합니다. RFC1766은 각각 ISO639, ISO3166을 참조하며, ISO Specification은 유료이므로 아래 웹 페이지를 참조합니다.
MSDN Language Codes: http://msdn.microsoft.com/en-au/library/ms533052(VS.85).aspx
자주 사용하는 국가코드는 en(영어), ja(일본어), zh-cn(중국어)이다.

B. <head>

meta, link, script, style, title 순서로 엘리먼트를 선언합니다.

<head>
<!-- meta -->
<meta http-equiv="Content-Type" content="text/html;charset=utf-8">
<meta http-equiv="Content-Script-Type" content="text/javascript">
<meta http-equiv="Content-Style-Type" content="text/css">
<!-- css -->
<link rel="stylesheet" type="text/css" href="css/default.css">
<!-- script -->
<script type="text/javascript" src="js/default.js"></script>
<!-- style -->
<style type="text/css">
[stuff]
</style>
<!-- title -->
<title>페이지명 > 사이트명</title>
</head>
C. <meta>

문서의 기본 인코딩, 뷰포트, 스크립트 형식, 스타일 형식 순서로 엘리먼트를 선언합니다. 뷰포트는 모바일 브라우저에 대응하는 HTML의 경우에만 선언합니다.

<meta http-equiv="Content-Type" content="text/html;charset=utf-8">
<meta name="viewport" content="width=device-width,initial-scale=1.0,maximum-scale=1.0,minimum-scale=1.0,user-scalable=no,target-densitydpi=medium-dpi">
<meta http-equiv="Content-Script-Type" content="text/javascript">
<meta http-equiv="Content-Style-Type" content="text/css">

HTML5 DTD 선언 시에는 다음과 같다.

<meta charset="utf-8">
<meta name="viewport" content="width=device-width,initial-scale=1.0,maximum-scale=1.0,minimum-scale=1.0,user-scalable=no,target-densitydpi=medium- dpi">
D. <link>

rel, type, href 순서로 애트리뷰트를 선언합니다.

<script type="text/javascript" src="js/default.js"></script>

script는 <head></head> 내에 선언하는 것을 원칙으로 합니다. 단, 성능상의 이슈가 있으면 개발 부서와 협의하여 선언 위치를 변경합니다.

참고
  • language 애트리뷰트는 HTML4 이전 버전의 하위 호환성을 위해 사용하는 비표준 애트리뷰트로, 사용하지 않습는다.
    HTML5 문서의 경우 <script>와 <style> 선언 시 type 애트리뷰트의 생략을 허용합니다.
E. <style>

type 애트리뷰트를 선언합니다.

<style type="text/css"> [stuff]
</style>
G. <title>

"메뉴 > 브랜드명 서비스"의 형식으로 작성합니다.

<title>소개 > 마음에이아이</title>
H. <body>

class 애트리뷰트는 웹페이지의 스킨셋을 변경해야 할 때 선택적으로 사용합니다.
모바일에서는 <body>에 브라우저별로 정의된 class명을 지정하고 이를 상속하여 브라우저별 CSS 렌더링 차이를 해결합니다. 변경 사항이 잦으니 아래 경로에서 확인합니다.
