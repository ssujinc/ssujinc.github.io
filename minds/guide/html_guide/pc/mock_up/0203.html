Style Guide 용어

A. 엘리먼트(Element)
HTML 문서를 구성하는 요소를 의미합니다. 일반적으로 시작 태그, 내용, 종료 태그로 구성됩니다.
B. 애트리뷰트(attribute)
H엘리먼트에 부여할 수 있는 특성을 의미합니다. 기본값이 설정되어 있으나 애트리뷰트를 선언하여 다른 값으로 설정할 수 있습니다.
C. 선택자(selector)
엘리먼트에 CSS 스타일을 적용하기 위한 패턴이다. 이 문서에서 언급하는 선택자의 종류는 다음 표와 표 1-1같다.
표 1-1 선택자 종류
이름 패던 예 설명
공통 선택자 * 어떤 엘리먼트와도 일치함.
타입 선택자 div 해당 엘리먼트와 일치함.
하위 선택자 div p 해당 엘리먼트의 하위의 모든 엘리먼트와 일치함.
자식 선택자 div > p 해당 엘리먼트의 자식 엘리먼트와 일치함.
Class 선택자 div.class 해당 엘리먼트의 class 애트리뷰트값과 일치함.
Id 선택자 div#id 해당 엘리먼트의 id 애트리뷰트값과 일치함.
D. 속성(property)
엘리먼트에 부여할 CSS 스타일 특성을 의미합니다. 기본값이 설정되어 있으나 속성을 선언하여 다른 값으로 설정할 수 있습니다. 각 속성 단위는 세미콜론(;)으로 구분지어 선언합니다.
E. 속성 값(value)
속성에 부여하는 값으로 콜론(:)으로 구분지어 선언합니다.
F. 예약어
이 문서에서 사용하는 예약어는 일반적인 의미와 다르게 사용됩니다. 이 문서에서 예약어는 객체, 이미지, 파일 및 폴더의 네이밍 시 의미를 일관되게 표현하기 위해 미리 지정해 놓은 일종의 언어 규칙을 의미합니다. 예를 들어, '검색'을 표현하는 예약어가 'srch'일 경우, 검색 영역을 위한 객체의 class 이름은 'srch'라는 예약어를 반드시 포함해야 합니다.

HTML 엘리먼트 작성 규칙

HTML 엘리먼트 작성 규칙은 반드시 선언해야 하는 애트리뷰트와 선택적 사용이 가능한 애트리뷰트에 대한 내용을 기술하고 선언 순서를 제시하여 코드 품질을 유지합니다.

CSS 코드 작성 규칙

A. 들여쓰기
CSS 코드는 들여쓰기를 하지 않습는다.
B. 공백
CSS 코드는 공백을 최소화합니다.
C. 빈 줄
CSS 코드의 빈 줄은 코드 그룹의 영역을 표시하기 위하여 사용합니다.
D. 줄 바꿈
CSS 코드의 가독성을 위한 줄 바꿈은 하지 않습는다.
E. 인코딩
CSS의 인코딩은 HTML의 인코딩과 동일하게 선언하여 HTML 문서가 브라우저에서 바르게 해석될 수 있도록 합니다.
F. 선택자
선택자 버그가 발생하지 않도록 사용 규칙을 준수합니다.
G. 속성
속성 선언 순서를 준수하여 개발자가 코드를 쉽게 파악할 수 있도록 하며, CSS 코드 최적화를 위해 속성 값을 축약하여 사용하고 약식 속 성을 허용 범위에 맞게 사용합니다.
H. z-index
z-index 속성 값을 범위에 맞게 사용하여 객체가 브라우저에서 바르게 표현되도록 합니다.
I. 핵
크로스 브라우징을 위해 제시된 핵(hack)에 한해 최소한의 사용을 허용합니다.
J. CSS 선언 타입
상황에 알맞은 CSS 선언 타입을 선택합니다.
K. 주석
CSS 코드의 주석은 코드 그룹을 구분하거나, 코드의 수정과 삭제를 표시하거나, 개발자가 참고해야 하는 사항을 기술합니다.
