참고사항

  • 작성된 마크업으로 준수하셔야 올바른 디자인을 제공 받을 수 있습니다.
  • 사용 하신 CSS에 따라 레이아웃 및 각 엘리먼트의 디자인이 자동으로 변경되니, 유의하시기 바랍니다.

DTD (Document Type Definition)

DTD 설정하지 않으면 브라우저에서 Quirks Mode로 인식하여 바르게 해석되지 않으므로 주의해야 합니다. DOCTYPE 앞에는 다름 문자가 오면 안됩니다. Code에서 DOCTYPE부분만 복사에서 써야 합니다. 
<!-- HTML5 -->
<!DOCTYPE html>

<!-- HTML 4.01 Transitional -->
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">

lang

HTML 문서가 어떤 언어로 작성 되었는지 알려주기 위해서 lang 속성을 이용해야 합니다. 화면 낭독 프로그램이 lang의 언어 타입에 따라 적합한 언어로 발음을 제공합니다. 
<!-- 한국어 -->
<html lang="ko">

<!-- 기타 -->
<html lang=en> <!-- 영어 -->
<html lang=ja> <!-- 일본어 -->
<html lang=zh-CN> <!-- 중국어(중국) -->
<html lang=zh-HK> <!-- 중국어(홍콩 특별 행정구, 중국) -->

Meta

<!-- charset -->
<meta charset="utf-8"/> <!-- 인코딩 방식 -->

<!-- X-UA-Compatible -->
<meta http-equiv="X-UA-Compatible" content="IE=edge"/> <!-- 브라우저의 문서모드를 지정. IE브라우저에서 최신모드로 지정된 DOCTYPE에 상관없이 IE8 이상 버전에서 항상 최신 표준 모드로 렌더링 -->

<!-- format-detection -->
<meta name="format-detection" content="telephone=no"/> <!-- 전화 번호의 모든 자동 서식을 제거 -->

<!-- viewport -->
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, minimum-scale=1.0, user-scalable=no"/> <!-- 모바일 브라우저 뷰포트 크기 조절 -->

OG Tag (Open Graph)

해당 웹페이지를 구성하는 여러 구조화된 정보들 예컨대 제목, 설명, 이미지 등을 아예 명시적으로 웹페이지쪽에서 직접 정해서 표기해준 것 입니다. 
<!-- Facebook Tag -->
<meta property="fb:app_id"           content="APP_ID" />

<!-- Open Graph Tag -->
<meta property="og:title"            content="웹 페이지 제목"/>
<meta property="og:type"             content="website"/><!-- 웹 페이지 타입 -->
<meta property="og:url"              content="웹 페이지 URL"/>
<meta property="og:image"            content="웹 페이지 대표 이미지"/>
<meta property="og:description"      content="웹 페이지 설명"/>

<!-- Twitter Tag -->
<meta property="twitter:card"        content="summary"/>
<meta property="twitter:url"         content="웹 페이지 URL"/>
<meta property="twitter:image"       content="웹 페이지 대표 이미지"/>
<meta property="twitter:title"       content="웹 페이지 제목"/>
<meta property="twitter:description" content="웹 페이지 설명"/>

icon & favicon

인터넷 웹 브라우저의 주소창에 표시되는 웹사이트나 웹페이지를 대표하는 아이콘입니다. 
<!-- icon (Internet Explorer & Windows Phone)  -->
<meta name="msapplication-square70x70logo"   content="resources/images/ico_favicon_70x70.png"><!-- smalltile -->
<meta name="msapplication-square150x150logo" content="resources/images/ico_favicon_150x150.png"><!-- mediumtile -->
<meta name="msapplication-wide310x150logo"   content="resources/images/ico_favicon_310x150.png"><!-- widetile -->

<!-- icon (Chrome & Opera) -->
<link rel="icon" sizes="192x192" href="resources/images/ico_favicon_192x192.png"><!-- 아이콘의 크기는 48px을 기본으로 합니다. 48px, 96px, 144px, 192px로 늘어납니다.-->

<!-- icon (Safari) -->
<link rel="apple-touch-icon" sizes="76x76"   href="resources/images/ico_favicon_76x76.png"/><!-- iPad -->
<link rel="apple-touch-icon" sizes="120x120" href="resources/images/ico_favicon_120x120.png"/><!-- iPhone Retina -->
<link rel="apple-touch-icon" sizes="152x152" href="resources/images/ico_favicon_152x152.png"/><!-- iPad Retina -->

<!-- icon favicon -->
<link rel="shortcut icon" type="image/x-icon" href="resources/images/ico_favicon_64x64.ico"/><!-- 윈도우즈 사이트 아이콘, HiDPI/Retina 에서 Safari 나중에 읽기 사이드바 -->

CSS (Cascading Style Sheets)

<ink rel="stylesheet" type="text/css" href="../../resources/css/font.css"/> <!-- WEB fonts -->
<link rel="stylesheet" type="text/css" href="../../resources/css/reset.css"/> <!-- 초기화 -->
